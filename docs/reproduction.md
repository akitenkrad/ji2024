## What is exact vs qualitative

- **Deterministic allocation/metrics path** — the allocation rule (S_queue + k-deferrals, no double-allocation, pool capacity respected) and all welfare/fairness metrics (SW, Avg r_size, Avg WT, Var r_size, Rop, co-Gini, F(V,NV)) are pure deterministic computations and reproduce bit-for-bit given a seed. The **policy ordering** is the quantitative-ish reproduction target: SW(`p_select`+`r_size`) ≥ SW(`r_random`).
- **LLM applicant behavior** — qualitative only. The local default model (`llama3.2`) differs from the paper's `gpt-3.5-turbo-0301`, so exact paper numbers are not expected.

## Policy-ordering check (offline, `--mock`)

With `--mock`, applicant decisions are scripted ("pick the first visible home"), so the SW ordering reflects the **deterministic allocation rule**, not real LLM behavior — that is expected. With a matched seed:

```bash
cargo run --release -- run --entry-condition p_select --resource-subset r_size   --runs 1 --seed 42 --mock   # SW ≈ +278.6
cargo run --release -- run --entry-condition p_select --resource-subset r_random --runs 1 --seed 42 --mock   # SW ≈ −73.0
```

SW(`r_size`) > SW(`r_random`) holds — the paper's key qualitative finding (size-sorted subsets yield higher welfare than random subsets). Because `r_size` presents larger homes first, the "pick first visible" mock obtains higher-utility allocations. Note: the `sweep` per-subset average uses *different* seeds per (entry × subset) combination, so for a clean comparison use matched single-run seeds as above.

> The absolute SW can be negative here: with the synthetic profile/pool distributions the rent-over-budget penalty can outweigh the size term. Only the *ordering* across policies is the target; the absolute level depends on the (unspecified) synthetic-environment parameters.

## Design uncertainties (paper §7)

- The synthetic environment (applicant count, resource count, profile distributions) is not specified in the paper body — externalised via CLI (`--n-applicants`, `--pool-ratio`, `--visible-subset-size`) with defaults (60 applicants, ratio 0.5).
- The POA predictor f̃, the GA hyperparameters (pool size, crossover/mutation rates) and the policy-vector decode are appendix-dependent — the `poa` subcommand is a **minimal Phase-3 stub** (fitness = one mock simulation; predictor f̃ and live-LLM fitness deferred).
