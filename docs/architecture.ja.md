# アーキテクチャ

SRAP-Agent は論文の Setup → Simulation → Evaluation を socsim の `WorldState` + 6-phase `Mechanism` ループへ写す．1 engine tick = 1 配分ラウンド．

## WorldState: `SrapWorld` (`simulation/src/world.rs`)

中央配分の世界 (グリッド / 網なし):

| フィールド | 型 | 役割 |
|-----------|-----|------|
| `applicants` | `BTreeMap<AgentId, Applicant>` | プロファイル (収入・家賃・家族・選好・記憶・active・脆弱層) |
| `pool` | `Vec<Resource>` | 希少資源プール (公共住宅: 面積・家賃・割当済みフラグ) |
| `policy` | `Policy` | π = (E_queue, S_queue, R_queue, m, k, c) |
| `queues` | `Vec<Queue>` | 入室条件で構成する m 本の待機キュー |
| `allocations` | `BTreeMap<AgentId, Option<ResourceId>>` | 応募者ごとの配分 R_j* |
| `metrics` | `SrapMetrics` | 当該ラウンドの満足度/公平性集計 |

`agent_ids()` は `BTreeMap` のキー (ソート済み `AgentId`) を返す → 決定論．応募者の `utility(resource)` は SW に使う決定論的な満足度である．

## ポリシー: `Policy` (`simulation/src/policy.rs`)

`Policy { entry_condition: {PBudget, PFamily, PSelect}, sort_strategy: {Fifo, Vfa, Vfr}, resource_subset: {RSize, RRent, RRandom}, m, k, c }`．

## 5 メカニズム × 6 フェーズ (`simulation/src/mechanisms.rs`)

| Mechanism | Phase | 役割 |
|-----------|-------|------|
| `PolicySetup` | Environment | プールの補充/確認，入室条件 E_queue でキュー q_1..q_m を構成 (PBudget/PFamily は収入/家族の中央値閾値; PSelect は全員入室を許し離脱は Decision に委ねる) |
| `ApplyDecision` | Decision | **★LLM 所在**．各 active 応募者が可視サブセット V(p_j) (R_queue でソートした上位 `visible_subset_size` 件) から希望住宅を選択，または ∅ = 離脱．同期: 希望を scratch に集め配分は次段で実行 |
| `AllocationRule` | Interaction | 決定論: S_queue で応募者を順序づけ (FIFO = AgentId 順; VFA = 脆弱層を属性で先頭; VFR = 家族/収入ランキングで脆弱層を先頭)，k-deferrals で割当 (第一希望 → 面積降順の次善を最大 k−1 件)．二重割当なし・プール容量厳守 |
| `EvaluateWelfare` | Reward | 配分済み応募者の outcome を累積し SW / Avg r_size / Avg WT / Var r_size / Rop / co-Gini / F(V,NV) を計算; `metrics.csv` 行を push |
| `UpdateMemory` | PostStep | 配分 → inactive; 未配分 → wait_time++ (プール枯渇なら離脱); 記憶サマリ追記; 枯渇 or 全員退出で `request_stop()` |

## 更新セマンティクスと scheduler

同期: 各ラウンド冒頭に active 応募者をスナップショットし，`ApplyDecision` で全希望を集めてから `AllocationRule` で一括配分する．scheduler は `RandomActivationScheduler` (到着パターンの確率性を engine RNG のシャッフルで担保); 配分順序自体は S_queue で決定論的に確定する．

## 二層 LLM (`simulation/src/llm.rs`)

`SrapClient = CachingClient<Box<dyn LlmClient>>`．本番 = `FallbackClient<OllamaClient, OpenAiClient>` を `Box<dyn LlmClient>` へ型消去．テスト / `--mock` は `socsim_llm::mock::ScriptedClient` を注入．LLM は `ApplyDecision` に閉じ込め，各ラウンド active 応募者 1 人あたり 1 回呼ぶ．

## POA — Policy Optimization Agent (`simulation/src/poa.rs`)

ポリシー空間上の遺伝的アルゴリズム外側ループ: 個体 = `Policy = (E_queue, S_queue, R_queue, m, k, c)`，トーナメント選択 + 一様交叉 + 遺伝子単位の突然変異 + 1-エリート保存．適応度 = 1 回の SRAP 配分実行を `f_pi(metrics, objective)` で評価し，決定論的 scripted mock (`FitnessKind::Mock`, オフライン・bit 決定論) かライブ LLM (`FitnessKind::Live { cache_path }`, Ollama→OpenAI + 世代間で共有する永続プロンプトキャッシュ) のいずれかで計算する．エリート保存により最良適応度は世代をまたいで単調非減少．

**予測器 `f̃`** (`Predictor`) はサロゲートモデルで，評価済みの `(ポリシー特徴ベクトル → 適応度)` サンプル上の重み付き k 近傍回帰 (再出現ポリシーには厳密一致キャッシュ) である．有効時 (`use_predictor`)，予測適応度が `incumbent_best − margin` を下回る個体は枝刈りされ，高価なシミュレーションをスキップしてサロゲート値を採用する．初期世代は必ず全評価してサロゲートを学習させ，枝刈り margin は初期適応度の散らばりからスケールする．`PoaResult` は `full_evals` と `evals_saved` を報告する．
