# CLI リファレンス

Rust バイナリは `srap` (`cargo build --release` で生成)．サブコマンド: `run`, `sweep`, `poa`, `reproduce`．

## 共通環境フラグ (`run`, `sweep`, `poa`)

| フラグ | 既定 | 意味 |
|--------|------|------|
| `--n-applicants` | 60 | 合成応募者数 |
| `--pool-ratio` | 0.5 | n_resources / n_applicants (希少性) |
| `--max-rounds` | 10 | ラウンド上限 (枯渇 / 全員退出で早期 stop) |
| `--visible-subset-size` | 5 | 可視資源サブセット V(p_j) のサイズ上限 |

> 合成環境パラメータ (応募者数・資源数・プロファイル分布) は論文本文に明示がない ([再現](reproduction.ja.md) 参照)．ここでは標準的な既定値で外部化する．

## `run` — 単一ポリシー

```bash
cargo run --release -- run \
    --entry-condition p_select --resource-subset r_size --queue-strategy fifo \
    --queues 3 --k 3 --c 2 --runs 10 --seed 42
```

| フラグ | 既定 | 値 / 意味 |
|--------|------|----------|
| `--entry-condition` | `p_select` | `p_budget` / `p_family` / `p_select` (E_queue) |
| `--resource-subset` | `r_size` | `r_size` / `r_rent` / `r_random` (R_queue) |
| `--queue-strategy` | `fifo` | `fifo` / `vfa` / `vfr` (S_queue) |
| `--queues` | 3 | m (キュー数) |
| `--k` | 3 | k-deferrals |
| `--c` | 2 | 選択キュー容量係数 |
| `--runs` | 1 | 独立試行数 (seed から派生) |
| `--seed` | random | コア RNG seed |
| `--temperature` | 0.0 | LLM 温度 |
| `--llm-seed` | 0 | LLM バックエンド seed |
| `--cache-path` | `.llm_cache/cache.json` | プロンプト→応答キャッシュ (`--mock` 時は無視) |
| `--output-dir` | `results` | 出力ベース |
| `--mock` | false | ライブ LLM の代わりに scripted mock (オフライン) |

出力 `results/{ts}/`: `config.json`, `metrics.csv` (long-format: run, round + 指標列), `llm_meta.json`．

## `sweep` — ポリシー因子の感度分析

```bash
cargo run --release -- sweep \
    --entry-conditions p_budget,p_family,p_select \
    --resource-subsets r_size,r_rent,r_random \
    --queue-strategies fifo --runs 30 --seed 42
```

出力 `results/{ts}_sweep/`: `sweep_summary.csv` (条件 × 試行ごとに最終 SW と全公平性指標), `metrics.csv`, `sweep_config.json`．

## `poa` — Policy Optimization Agent

```bash
cargo run --release -- poa --objective satisfaction \
    --iterations 50 --pool-size 50 --use-predictor --seed 42 --mock
```

| フラグ | 既定 | 意味 |
|--------|------|------|
| `--objective` | `satisfaction` | `satisfaction` / `fairness` (f(π) の重み) |
| `--iterations` | 20 | GA 世代数 M |
| `--pool-size` | 12 | 個体群サイズ |
| `--mutation-rate` | 0.2 | 遺伝子単位の突然変異率 |
| `--tournament-size` | 3 | トーナメントサイズ |
| `--use-predictor` | off | サロゲート `f̃` でフル評価を枝刈りする |
| `--mock` | off | scripted-mock 適応度 (オフライン)．外すと live-LLM 適応度 |
| `--cache-path` | `.llm_cache/cache.json` | 永続プロンプトキャッシュ (live 時; 世代間で共有) |
| `--seed` | 42 | RNG seed |

適応度 = 1 回の SRAP 配分実行を `f_pi(metrics, objective)` で評価する．出力 `results/{ts}_poa/`: `poa_history.csv` (世代・最良/平均適応度・累積 `full_evals` / `evals_saved`・最良ポリシー), `poa_config.json`．最良適応度は単調非減少 (エリート保存)．`--use-predictor` ではサロゲートが «現行エリートに勝てない» 評価をスキップする．

## `reproduce` — Table 2/3 + Fig.4

```bash
cargo run --release -- reproduce --mock --seed 42      # 高速スモークは --quick
```

| フラグ | 既定 | 意味 |
|--------|------|------|
| `--runs` | 10 | matched-seed 条件あたり試行数 (Table 2) |
| `--poa-iterations` | 20 | POA 世代数 (Table 3 / Fig.4) |
| `--poa-pool-size` | 16 | POA 個体群サイズ |
| `--mock` | off | scripted-mock 評価 (オフライン)．外すと live LLM |
| `--quick` | off | POA 予算・応募者数を縮小して高速化 |
| `--seed` | 42 | RNG seed |

Table 2 (matched-seed paired の社会的厚生順序; 論文の «r_size 最高 / r_random 最低» を paired win-rate で報告)・Table 3 (POA 最適化 `π_s*` / `π_f*`)・Figure 4 (POA 収束) を再現する．出力 `results/reproduce_{ts}/`: `table2_sw_by_policy.csv`・`table3_optimized_policies.csv`・`poa_history_<objective>.csv`・`reproduce_summary.json` (観測 vs 論文知見の PASS / off-anchor)．図は `uv run srap-tools reproduce --results-dir results/latest` で生成する．
