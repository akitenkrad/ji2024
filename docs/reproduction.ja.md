## 厳密 vs 定性

- **決定論的な配分/指標パス** — 配分規則 (S_queue + k-deferrals; 二重割当なし・プール容量厳守) と全満足度/公平性指標 (SW, Avg r_size, Avg WT, Var r_size, Rop, co-Gini, F(V,NV)) は純決定論計算で seed が与えられれば bit 単位で再現する．**ポリシー順序** が定量的な再現目標である: SW(`p_select`+`r_size`) ≥ SW(`r_random`)．
- **LLM 応募者挙動** — 定性のみ．ローカル既定モデル (`llama3.2`) は論文の `gpt-3.5-turbo-0301` と異なるため論文値との厳密一致は期待しない．

## ポリシー順序チェック (オフライン, `--mock`)

`--mock` では応募者の意思決定が scripted (「最初の可視 home を選ぶ」) なので，SW の順序は **決定論的配分規則** を反映し，実 LLM 挙動ではない (これは想定どおり)．同一 seed で:

```bash
cargo run --release -- run --entry-condition p_select --resource-subset r_size   --runs 1 --seed 42 --mock   # SW ≈ +278.6
cargo run --release -- run --entry-condition p_select --resource-subset r_random --runs 1 --seed 42 --mock   # SW ≈ −73.0
```

SW(`r_size`) > SW(`r_random`) が成り立つ — 論文の核心知見 (面積でソートしたサブセットはランダムより高厚生)．`r_size` は広い住宅から提示するため，「最初の可視 home を選ぶ」mock は高効用の割当を得る．注: `sweep` のサブセット別平均は (入室 × サブセット) ごとに *異なる* seed を使うため，クリーンな比較には上記のように単一実行で seed を揃える．

> 絶対 SW は負になりうる: 合成プロファイル/プール分布では家賃超過ペナルティが面積項を上回ることがある．目標はポリシー間の *順序* のみであり，絶対水準は (明示されていない) 合成環境パラメータに依存する．

## 設計上の不確実性 (論文 §7)

- 合成環境 (応募者数・資源数・プロファイル分布) は論文本文に明示がない — CLI (`--n-applicants`, `--pool-ratio`, `--visible-subset-size`) で外部化し既定値 (応募者 60, 比 0.5) を置く．
- GA ハイパーパラメータ (プール規模・交叉/突然変異率)・予測器 `f̃` の margin は付録依存で，CLI フラグとして標準既定値で外部化する．`poa` サブコマンドは決定論 mock かライブ LLM のいずれかで適応度を評価し，予測器 `f̃` (最近傍サロゲート) がフル評価を枝刈りする．付録値は論文本文にないため，SW の *順序* と POA の *改善方向* (絶対適応度ではない) を再現目標とする．

## 一括再現 (`reproduce`)

```bash
cargo run --release -- reproduce --mock --seed 42      # 高速スモークは --quick
uv run srap-tools reproduce --results-dir results/latest   # 図を生成
```

`reproduce` は Table 2 を **matched-seed paired** 比較 (同一環境で資源サブセットのみを変える) として実行し，SW(`r_size`) ≥ SW(`r_random`) が成立した環境の割合 (paired win-rate) を報告する — 合成 rent ペナルティで per-environment の SW がノイジーなため，独立シードの平均比較より paired win-rate の方が頑健である．続けて両目標で POA を実行し (Table 3 `π_s*` / `π_f*`)，目標別の収束履歴 (Fig.4) を出力する．PASS 基準は論文の核心知見 (`r_size` 最高 / `r_random` 最低) + POA の非減少トレンドで，`reproduce_summary.json` がチェックごとに観測 vs 論文知見を記録する．

## 実 Ollama スモーク (メインセッション用)

```bash
ollama pull llama3.2:latest
export OLLAMA_HOST=http://localhost:11434 OLLAMA_MODEL=llama3.2:latest
# コストを抑えるため小さく: 約 n_applicants × rounds 回の LLM 呼び出し (cache で再実行は無料)．
cargo run --release -- run --entry-condition p_select --resource-subset r_size \
    --queues 3 --k 3 --c 2 --runs 1 --seed 42 --n-applicants 12 --max-rounds 3
```
